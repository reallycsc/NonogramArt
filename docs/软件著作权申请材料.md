### 1.3 开发目的

本软件的开发目的是为移动端用户提供一款寓教于乐的数织解谜游戏，填补国内市场在数织类游戏中缺乏中国文化知识内容的空白。通过将数织这一经典逻辑解谜玩法与中国传统文化、自然科学、艺术鉴赏等知识内容相结合，为用户提供在休闲娱乐中学习和探索知识的全新体验。

具体目标包括：

1. **玩法创新**：将传统的数织玩法与图片拼图机制相结合，创造"数织还原→图片收集→画册点亮"的独特游戏循环；
2. **知识传播**：通过66本涵盖7大知识领域的主题画册，传播人文历史、艺术创作、自然地理等领域的知识；
3. **文化传承**：采用中国风卡通视觉风格，弘扬中华优秀传统文化；
4. **普适设计**：设计从5×5到25×25五种难度级别，满足从儿童到成人的全年龄段用户需求。

数织艺术软件的主要功能包括：

#### 1.5.1 数织解谜功能（核心功能）

- **关卡挑战**：玩家根据行列提示数字，通过逻辑推理填充数织网格

3. 自研数织生成器，基于Otsu动态阈值、排名填充法和复杂度分析，从图片自动生成多难度数织关卡；
4. 采用中国风卡通视觉风格，水墨风格UI主题，支持横竖屏自适应切换；
5. 实现了DLC分包下载机制，支持大规模游戏资源的按需加载。

## 二、软件说明书

### 2.1 软件概述

数织艺术是一款数织解谜类休闲益智游戏。游戏以"书架-画册-图片-关卡"四级层级结构组织内容，玩家从书架中选择主题画册，浏览画册中的图片，选择图片的数织关卡进行挑战，完成所有关卡后拼合完整图片，形成"拼图点亮"的仪式感。

软件包含以下核心内容：

| 统计项 | 数量 |
|--------|------|
| 书架总数 | 7个 |
| 画册总数 | 66本 |
| 图片总数 | 4650张 |
| 谜题总数 | 28819个 |

#### 2.2.1 主菜单模块

主菜单为游戏的入口界面，包含以下功能：

- **开始旅程**：进入书架界面，选择画册开始游戏
- **设置**：调整音量、语言等游戏设置
- **背景展示**：展示中国风水墨动画背景

#### 2.2.2 书架模块

书架界面展示7大知识领域的主题书架，每个书架下包含多本画册：

| 书架名称 | 画册数量 | 包含画册举例 |
|----------|----------|-------------|
| 人文历史 | 12本 | 中国通史、世界通史、亚洲文明史、欧洲文明史等 |
| 艺术创作 | 10本 | 中国书画艺术、西方绘画艺术、雕塑艺术等 |
| 自然地理 | 8本 | 山脉与高原、河流与湖泊、地质地貌等 |
| 生物世界 | 10本 | 哺乳动物、鸟类、鱼类、昆虫等 |
| 生活社会 | 11本 | 美食、节日庆典、体育竞技等 |
| 科技工业 | 11本 | 天文学、信息技术、数学与物理等 |
| 综合素材 | 4本 | 抽象图案、符号标志、纹理材质等 |

画册按钮显示状态：
- **未解锁**：灰色封面，显示锁图标
- **进行中**：灰色封面，显示完成百分比
- **已完成**：彩色封面，显示"已完成"标签
- **DLC未下载**：灰色封面，显示下载图标

支持左右翻页切换书架，支持滑动手势操作。

#### 2.2.3 画册详情模块

画册详情界面展示当前画册的所有图片，以网格形式排列：

- **图片卡片**：显示图片缩略图、标题、完成状态
- **完成状态**：已完成的分块显示彩色，未完成的显示剪影
- **进度指示**：显示当前图片的完成百分比
- **图片详情**：点击图片查看完整图片和分块选择

#### 2.2.4 数织游戏模块

数织游戏是软件的核心玩法模块，界面布局如下：

- **顶部**：分块名称、返回按钮、提示按钮

- **上方**：列提示数字
- **中部**：数织网格（支持5×5到25×25五种尺寸）

- **填充模式**：点击或拖拽填充格子
- **标记模式**：点击或拖拽标记空白格（×标记）
- **缩放**：双指缩放查看大网格
- **拖拽**：单指拖拽移动视图

难度分级：

### 2.3 操作流程说明

#### 2.3.1 游戏主流程

```
启动游戏 → 主菜单 → 点击"开始旅程"
    → 书架界面 → 选择书架 → 选择画册
    → 画册详情 → 选择图片 → 选择分块
    → 数织游戏 → 完成关卡 → 返回画册详情
    → 完成所有分块 → 图片拼合完成 → 解锁下一张图片
    → 完成所有图片 → 画册完成 → 解锁下一本画册
```

#### 2.3.2 数织游戏操作流程

```
进入数织关卡 → 查看行列提示
    → 填充模式：点击/拖拽填充格子
    → 标记模式：点击/拖拽标记空白格
    → 填充错误：红色闪烁提示，扣除生命值
    → 完成关卡：展示还原的分块图片 + 恭喜动画
    → 返回画册详情
```

#### 2.3.3 画册完成动画流程

```
完成画册内所有图片的数织关卡
    → 返回书架界面
    → 画册按钮从灰色渐变为彩色（首次触发）
    → 播放恭喜音效
    → 标记动画已播放（后续不再触发）
```

### 2.4 技术架构说明

#### 2.4.1 系统架构

软件采用Godot引擎的Autoload单例模式管理全局状态，主要模块包括：

| 模块 | 职责 |
|------|------|
| GameManager | 游戏状态管理、进度存档、画册解锁、完成度计算 |
| NonogramManager | 数织游戏逻辑、关卡加载、游戏状态管理 |
| AudioManager | 音频管理、背景音乐切换、音效播放 |
| OrientationManager | 横竖屏自适应切换 |
| DLCManager | DLC分包下载管理 |
| ToastManager | 提示信息管理 |
| AnimationManager | 动画效果管理 |

#### 2.4.2 数据层

| 数据文件 | 内容 |
|----------|------|
| albums.json | 书架与画册定义（7个书架、66个画册） |
| pictures/{album_id}.json | 每个画册的图片数据（68个文件） |
| puzzles/{album_id}/{puzzle_id}.json | 数织关卡数据（约15000+个文件） |

#### 2.4.3 核心算法

**数织求解器（NonogramSolver）**：
- 采用约束传播（Constraint Propagation）算法
- 对每行/列调用line_solve确定必填和必空格子
- 支持深度试探（Probing）增强推理能力
- 确保所有关卡均可通过纯逻辑推理求解

**数织生成器**：
- 基于Otsu动态阈值算法确定亮度分割点
- 采用排名填充法生成二值化网格
- 动态填充率计算（25%~65%范围）
- 空满行自动修复
- 提示叉叉自动生成（标记空白格辅助推理）
- 可推理性自动验证

3. **开发完成日期**：不得晚于当前日期

6. **功能-代码一致性**：说明书描述的功能必须在源代码中有对应实现
