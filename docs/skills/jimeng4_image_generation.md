# 即梦4.0图片生成Skill

## 概述

使用即梦(Jimeng) CLI工具的4.0模型，批量生成画册配图，并生成对应的像素图。支持防爬限流、内容审核自动重试、进度断点续传。完整流程：原图生成 → 像素图生成 → 数据文件更新。

## 核心规则：每主题生成4张图片

### 批量生成规则

| 规则 | 说明 |
| ---- | ---- |
| **生成数量** | 每个主题/场景使用同一套提示词生成4张图片 |
| **后缀命名** | 文件后缀添加对应数字 `_1`、`_2`、`_3`、`_4` |
| **顺延机制** | 多次生成同一主题时，数字顺延（如已存在`_1`到`_4`，则新增`_5`到`_8`） |
| **不删除旧图** | 重新生成不删除原有图片文件，只在最高序号后追加新图片 |

### 命名规范

```
原图：{主题ID}.jpg
4张图：{主题ID}_1.jpg、{主题ID}_2.jpg、{主题ID}_3.jpg、{主题ID}_4.jpg
顺延：{主题ID}_5.jpg、{主题ID}_6.jpg、{主题ID}_7.jpg、{主题ID}_8.jpg
```

### 文件存在性检查逻辑

```python
def get_next_suffix(topic_id, base_dir):
    """获取下一个可用的文件后缀数字"""
    existing = []
    for f in base_dir.glob(f"{topic_id}_*.jpg"):
        suffix = f.stem.split('_')[-1]
        if suffix.isdigit():
            existing.append(int(suffix))
    
    if not existing:
        return 1  # 首次生成，从_1开始
    
    max_num = max(existing)
    return max_num + 1  # 顺延到下一组
```

## 提示词规则

### 提示词结构

```
{风格前缀}{构图指导词}，{场景描述}，{配色关键词}{风格后缀}
```

### 风格前缀（固定）

```
中国风卡通插画，圆润流畅线条，色彩丰富多样，画面构图饱满，
```

> **关于Q版人物规则（v7变更）**：v6保留"Q版2-3头身比例人物"但配合构图指导词仍无法避免"一排角色正面站立排列"问题。v7从风格前缀中**移除"Q版2-3头身比例人物"**，因为该词是导致AI默认生成"人物展示图"式排列的根本原因——AI看到"Q版人物"后优先画人物，忽略构图和环境。

### 构图指导词（随机选取，每张图不同）

**关键改进**：在风格前缀和场景描述之间插入构图指导词，打破AI默认的"正面平视+均匀排列"构图模式。

```python
import random

COMPOSITION_TEMPLATES = [
    "远景大场景，俯瞰视角，人物融入环境",
    "动态场景，侧面视角，人物正在行动中",
    "近景特写，聚焦核心事件",
    "广角镜头，环境氛围浓厚，人物在场景中",
    "叙事性构图，前景中景后景层次分明",
    "斜角构图，人物与建筑交错",
    "纵深构图，近大远小，场景有纵深感",
]

def build_prompt(scene):
    composition = random.choice(COMPOSITION_TEMPLATES)
    return f"{STYLE_PREFIX}{composition}，{scene}，{SHELF_COLOR_KEYWORDS}{STYLE_SUFFIX}"
```

> **为什么需要构图指导词**：即梦4.0模型在看到"Q版人物"后，默认构图是"正面平视+均匀排列"，导致大量图片像人物展示图。添加构图指导词后，AI会根据指导词调整视角、景别和人物布局，使画面更像历史场景插画。

### 配色关键词

**重要：不要使用固定配色关键词（如"赭石色水墨灰金色点缀"），这会导致所有图片变成同一色系。**

| 书架 | 配色关键词 |
| ---- | ---- |
| 人文历史 | 色彩搭配和谐 |
| 艺术创作 | 色彩搭配和谐 |
| 自然地理 | 色彩搭配和谐 |
| 生物世界 | 色彩搭配和谐 |
| 生活社会 | 色彩搭配和谐 |
| 科技工业 | 色彩搭配和谐 |
| 综合素材 | 色彩搭配和谐 |

> 经验教训：之前使用"赭石色水墨灰金色点缀"导致90%图片为暖红色调。去掉固定配色后，不同场景（森林、沙漠、海洋）能自然呈现不同色调。让场景描述本身决定色调，而非强制配色。

### 风格后缀（固定）

```
，画面边缘有丰富细节，不要大面积纯色背景，绝对不要出现任何中文文字或汉字，不要出现任何英文字母或单词，no text,no letters,no characters,no writing,no calligraphy,no English words,no English letters,no symbols
```

> **关于中文乱码问题**：即梦4.0模型对中文否定指令理解较弱，仅用中文说"不要出现中文文字"效果有限。v6在后缀中**追加英文否定词** `no text,no letters,no characters,no writing,no calligraphy`，中英文双重约束，大幅减少乱码文字的出现。v8进一步增强约束：追加"绝对不要出现任何中文文字或汉字，不要出现任何英文字母或单词"和"no English words,no English letters,no symbols"，完全禁止任何文字出现。

> **关于全黑背景问题**：虽然后缀包含"不要大面积纯色背景"，但AI仍可能生成全黑背景。解决方案是在**场景描述中明确添加具体环境元素**，如"热带雨林背景，阳光明媚，绿色植被环绕"、"蓝天白云下"、"星空璀璨的夜晚"、"晴朗天空下的波斯帝国御道，两侧绿洲与驿站在阳光中延伸至远方"等，让AI有具体的环境细节可绘制，而非默认填充黑色。

### 负面提示词

```
写实照片,3D渲染,暗色调,灰暗,恐怖,血腥,真实人物,photorealistic,dark,gloomy,写实风格,摄影,中文文字,汉字,书法字,文字水印,text,letters,characters,writing,calligraphy,一排人物站立,正面排列,人物展示图,symbols,markers,signs,signage
```

> **v6新增**：负面提示词中增加了 `text,letters,characters,writing,calligraphy`（英文反文字词）和 `一排人物站立,正面排列,人物展示图`（反排列构图词），从负面约束角度进一步减少乱码和排列问题。v8进一步增加了 `symbols,markers,signs,signage`（反符号词），完全禁止任何符号或标记出现。

### CLI命令格式

```bash
dreamina.exe text2image \
  --prompt="{完整提示词}" \
  --ratio=3:2 \
  --model_version=4.0 \
  --resolution_type=2k \
  --poll=120
```

### CLI返回格式

```json
{
  "submit_id": "uuid",
  "gen_status": "success",
  "result_json": {
    "images": [
      {
        "image_url": "https://...",
        "width": 2496,
        "height": 1664
      }
    ]
  }
}
```

### gen_status状态说明

| 状态 | 说明 | 处理方式 |
| ---- | ---- | ---- |
| success | 生成成功 | 下载图片 |
| fail | 生成失败 | 可能是内容审核，尝试安全提示词 |
| running/pending/queued | 排队中 | 用query_result轮询 |
| （无输出/JSON解析失败） | CLI异常 | 等待后重试 |

### ⚠️ 重要问题：一排角色正面站立排列

**问题现象**：大量图片出现"3-6个Q版角色正面面对镜头站立/坐成一排"的构图模式，看起来像"人物展示图"而非"历史场景插画"。

**根本原因分析**：

| 问题点 | v5提示词 | AI理解 |
| ------ | ---------- | ------ |
| **"Q版2-3头身比例人物"** | 风格前缀包含 | AI优先画Q版人物 → 人物展示图式排列 |
| **缺少构图词** | 无 | 默认正面平视均衡构图 → 角色均匀排列 |
| **缺少视角词** | 无 | 默认正面平视 → 角色正面面对镜头 |
| **缺少动作词** | 无 | 默认静态姿势 → 站立不动 |
| **场景描述类型** | 叙述性文字（谁做了什么） | "画这些人" → 一排人物 |

**v6解决方案（效果不足）**：

1. **保留"Q版2-3头身比例人物"**，但添加随机构图指导词打破默认排列模式
2. **7种构图模板随机选取**：远景俯瞰、动态侧面、近景特写、广角环境、叙事层次、斜角交错、纵深近大远小
3. **负面提示词增加**：`一排人物站立,正面排列,人物展示图`
4. **实测结果**：构图指导词+负面排列词仍无法克服"Q版人物"的强引导作用，AI依然默认生成一排站立人物

**v7解决方案（当前）**：

1. **从风格前缀中移除"Q版2-3头身比例人物"**——这是根本原因，AI看到"Q版人物"后优先画人物展示图
2. 保留构图指导词和负面排列词作为辅助约束
3. "中国风卡通插画"风格本身已能保证画面风格统一，无需额外强调Q版人物

### ⚠️ 重要问题：中文乱码字符

**问题现象**：部分图片中出现混乱的中文/日文字符，如"世界"、"の"等。

**根本原因**：即梦4.0模型对中文否定指令理解较弱，仅用中文说"不要出现中文文字"效果有限。

**v6解决方案（已实施）**：

1. **后缀追加英文否定词**：`no text,no letters,no characters,no writing,no calligraphy`
2. **负面提示词增加英文反文字词**：`text,letters,characters,writing,calligraphy`
3. **中英文双重约束**，大幅减少乱码文字出现

### ⚠️ 重要问题：部分图片仍有大量文字（中文或英文）

**问题现象**：部分图片（特别是涉及文化、文字、数学、公告等主题）中仍会出现大量文字，包括：
- 中文字符（乱码或有意义的文字）
- 英文字母和单词
- 符号和标记

**根本原因分析**：
1. **即梦4.0对文字限制不够严格** - 即使有"no text"，AI仍可能在某些场景（如数学成就、公告栏、文字展示等）中生成文字
2. **场景描述触发文字联想** - 当场景描述包含"数学"、"公告"、"诏书"等词汇时，AI可能自然联想到需要文字展示
3. **缺乏更强的否定词** - "不要出现中文文字"不够强力

**v8解决方案（已实施）**：

1. **增强中文否定词强度**：将"不要出现任何中文文字或汉字"改为"**绝对不要出现任何中文文字或汉字**"
2. **明确禁止英文**：追加"**不要出现任何英文字母或单词**"
3. **追加更多英文否定词**：后缀追加 `no English words,no English letters,no symbols`
4. **负面提示词增加反符号词**：追加 `symbols,markers,signs,signage`
5. **场景中添加具体环境描述**：除了禁止文字外，还在场景描述中添加具体环境（如"晴朗天空下的波斯帝国御道，两侧绿洲与驿站在阳光中延伸至远方"），让AI专注于环境和场景，而非文字展示

**测试验证**：重新生成有问题的5张图片（039、043、051、068、072），均未出现文字问题，画面质量良好。

## 像素图生成

原图生成完成后，需要为每张图片生成对应的像素图版本，作为数织关卡完成后的填充模板使用。

### 像素图规则

| 规则 | 说明 |
| ---- | ---- |
| 命名规范 | 像素图文件名 = 原图文件名 + `_pixel.jpg` |
| 输出目录 | 与原图在同一目录，不单独创建子目录 |
| 分辨率 | 与原图相同分辨率（2496×1664） |
| 生成方式 | 根据数织关卡的网格数目生成对应的像素数量，不使用solution |
| 放大方式 | 使用 NEAREST 插值保持像素风格 |
| 文件格式 | JPG（quality=95） |

### 像素图尺寸计算

根据图片的分块配置（3×2）和每块的网格大小（N×N）计算小像素图尺寸：

| grid_size | 难度 | 小像素图尺寸 | 放大后尺寸 |
| --------- | ---- | ------------ | ---------- |
| 5 | tutorial | 15×10 | 2496×1664 |
| 10 | easy | 30×20 | 2496×1664 |
| 15 | medium | 45×30 | 2496×1664 |
| 20 | hard | 60×40 | 2496×1664 |
| 25 | expert | 75×50 | 2496×1664 |

> 计算公式：小像素图宽度 = 3 × grid_size，小像素图高度 = 2 × grid_size

### 像素图生成流程

1. 打开原图，获取宽高（img_width × img_height）
2. 计算小像素图尺寸：pixel_w = 3 × grid_size，pixel_h = 2 × grid_size
3. 计算每个分块的采样区域：block_width = img_width / 3，block_height = img_height / 2
4. 对每个像素点，计算其在原图中的采样坐标：
   - sample_x = x_block × block_width + block_width × (x_pixel + 0.5) / grid_size
   - sample_y = y_block × block_height + block_height × (y_pixel + 0.5) / grid_size
5. 从原图采样该坐标的颜色，写入小像素图
6. 使用 NEAREST 插值将小像素图放大到原图分辨率
7. 保存为JPG格式（quality=95）

### 采样坐标说明

采样时使用 `(x_pixel + 0.5) / grid_size` 确保采样点位于每个网格单元的中心，而非边缘。这样采样的颜色最能代表该网格单元的视觉内容。

```
原图 (2496×1664)                    小像素图 (30×20, grid=10)
┌──────────┬──────────┬──────────┐   ┌────┬────┬────┐
│          │          │          │   │ 0,0│ 1,0│ 2,0│  ← 每格采样中心颜色
│  block0  │  block1  │  block2  │   ├────┼────┼────┤
│ (832×832)│ (832×832)│ (832×832)│   │ 0,1│ 1,1│ 2,1│
│          │          │          │   ├────┼────┼────┤
├──────────┼──────────┼──────────┤   │ 0,2│ 1,2│ 2,2│
│          │          │          │   └────┴────┴────┘
│  block3  │  block4  │  block5  │
│ (832×832)│ (832×832)│ (832×832)│   NEAREST放大
│          │          │          │   → 2496×1664
└──────────┴──────────┴──────────┘
```
