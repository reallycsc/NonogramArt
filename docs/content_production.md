# 《数织艺术》项目内容生产文档

> 版本 1.0 | 2026-05-07

***

## 一、项目内容概览

一款数织解谜游戏，玩家通过完成数织关卡收集图片分块，最终将拼图拼合完整，点亮书架中的所有画册。游戏核心循环：选择主题画册 → 查看画册图片 → 选择图片分块 → 解数织关卡 → 完成图片收集 → 解锁下一张图片 → 完成主题画册 → 解锁下一本画册。

通过数织关卡还原的图片分块，是画册图片的一块拼图。完成所有拼图后，拼合成完整的高清图片，形成"拼图点亮"的仪式感。

所有画册图片均采用 **中国风卡通风格**，具有以下统一特征：

| 特征       | 描述             |

| **线条风格** | 圆润流畅，略带书法笔触感   |

| **场景构建** | 层次分明，富有故事性     |

### 分类使用核心规则

**交叉主题优先级**：
艺术创作 > 人文历史 > 生活社会 > 自然地理 > 生物世界 > 科技工业

例：故宫建筑图优先归入《建筑艺术》，而非《中国文化遗产》；梵高《向日葵》优先归入《西方绘画艺术》，而非《花卉》

**主体优先原则**：
图片中最突出的主体是什么，就归入对应的画册

例：有人物的风景照，若风景是主体则归入自然地理，若人物是主体则归入生活社会

### 难度分布

| 难度 | 网格大小  | 说明     | 使用场景                   |
| -- | ----- | ------ | ---------------------- |
| 入门 | 5×5   | 新手引导专用 | 仅《中国通史》前几章及部分简单主题画册    |
| 简单 | 10×10 | 基础难度   | 自然地理、生物世界、生活社会等简单主题    |
| 中等 | 15×15 | 中等难度   | 艺术作品、历史场景等中等复杂度主题      |
| 困难 | 20×20 | 较高难度   | 复杂历史场景、建筑艺术等较高复杂度主题    |
| 专家 | 25×25 | 最高难度   | 战争军事、科技工业、精细艺术等最高复杂度主题 |

### 难度配置原则

1. **图片复杂度匹配**：图片内容越复杂，适用难度越高
   - 简单图片（单一主体、简洁线条）：5×5\~15×15
   - 中等复杂度图片（中等细节、多个元素）：10×10\~20×20
   - 高复杂度图片（丰富细节、复杂场景）：15×15\~25×25
2. **新手引导专属**：5×5难度仅出现在《中国通史》前几章，作为新手引导关卡
3. **难度递增**：同一画册内难度随进度逐步提升，从简单到困难

## 五、内容生产工具说明

### 5.1 配图生成流程

1. `generate_jimeng_images.py` — 根据画册的 `image_grid` 配置（X×Y）计算宽高比，调用AI生成配图
2. 图片自动保存到对应画册目录，保留原始分辨率

### 5.2 谜题生成流程

1. `generate_puzzles_from_image.py` — 从完整图片按 X×Y 等分拆分为分块
2. 对每个分块进行二值化处理，生成数织网格
3. 自动验证可推理性，必要时添加 hint\_cells

### 5.3 验证与修复工具

## 七、图片生成规则（核心约束）

生成的画册图片必须遵循以下规则：

| 规则        | 说明                                                      |
| --------- | ------------------------------------------------------- |
| **完整性**   | 图片本身是一张完整的艺术作品，包含主题场景和所有核心元素                            |
| **可拆分性**  | 图片在逻辑上可被划分为 X×Y 个大小相同的矩形区域（每个区域对应一个数织关卡）                |
| **难度匹配**  | 每个区域的数织网格大小为 N×N，N 值必须符合当前难度阶段（简单:10、中等:15、困难:20、专家:25） |
| **可变性**   | 不同图片的 X 和 Y 可以不同，根据内容灵活设计                               |
| **可推理性**  | 从每个区域提取的数织关卡必须能通过纯逻辑推理完成，无需猜测                           |
| **不拉伸**   | 图片必须以正确的宽高比直接生成，禁止生成后再拉伸/缩放变形                           |
| **风格一致性** | 所有图片必须采用一致的卡通风格生成，保持色彩、线条、造型风格统一                        |

### 6.1 像素图生成规则

每张画册图片需要生成对应的像素图版本，规则如下：

| 规则       | 说明                              |
| -------- | ------------------------------- |
| **命名规范** | 像素图文件名 = 原图文件名 + `_pixel.png`   |
| **输出目录** | 与原图在同一目录，不单独创建子目录               |
| **分辨率**  | 与原图相同分辨率                        |
| **生成方式** | 根据数织关卡的网格数目生成对应的像素数量，不使用solution |
| **放大方式** | 使用 NEAREST 插值保持像素风格             |

**像素图生成流程：**

1. 根据图片的分块配置（X×Y 分块）和每块的网格大小（N×N）计算小像素图尺寸
2. 创建 (N×X) × (N×Y) 的小像素图（例如：3×2分块，每块5×5网格 → 15×10小像素图）
3. 使用 NEAREST 插值放大到与原图相同的分辨率
4. 像素图作为数织关卡完成后的填充模板使用

### 6.2 全满/全空行/列预防策略

为提高数织关卡质量，生成时需采取以下预防措施：

| 策略        | 说明                           |
| --------- | ---------------------------- |
| **提示词优化** | 在提示词中加入"画面边缘有丰富细节，不要大面积纯色背景" |
| **全空行修复** | 自动检测并在空白行中间添加像素点             |
| **全满行修复** | 自动检测并在满行边缘移除像素点              |
| **居中裁剪**  | 图片分块时采用居中裁剪，避免边缘纯色区域         |

**图片布局规范：**

- 图片宽高比必须与 X:Y 匹配（如 X=3, Y=2 时宽高比为 3:2）
- 生成图片时，通过 API 的 size 参数指定匹配宽高比的分辨率，而非生成后 resize
- 支持的宽高比与对应 API 尺寸：1:1(2048x2048)、4:3(2304x1728)、3:4(1728x2304)、16:9(2848x1600)、9:16(1600x2848)、3:2(2496x1664)、2:3(1664x2496)、21:9(3136x1344)
- 图片包含 X×Y 个核心分块，每个分块占据一个独立的区域
- 分块区域边界清晰，便于自动提取和数织关卡生成
- 各区域在视觉上构成完整的场景，而非孤立的图标集合

### 8.3 生成流程规范

1. **使用统一模板**：所有音乐生成必须使用上述提示词格式
2. **时长统一**：统一设置时长为180秒（3分钟）
3. **格式规范**：选择instrumental模式，确保无 vocals
4. **风格一致性**：同一书架的音乐建议批量生成，确保风格过渡自然
5. **质量检查**：生成后检查是否符合意境描述，不符合则调整提示词重新生成
